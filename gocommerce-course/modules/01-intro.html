Module 1

The Checkout Journey

Have you ever wondered what happens under the hood when a user clicks "Buy" on a static website? Static sites can't securely process payments or manage inventory on their own.

That's where GoCommerce steps in. Think of GoCommerce as a highly efficient checkout clerk at a specialized boutique. The clerk coordinates between the customer, the bank, and the stockroom without ever moving from the register.

The Bridge

GoCommerce is an API that bridges the gap between static pages (like a Gatsby or Next.js site) and dynamic backend services (like databases and payment gateways).

Let's look at the main structure that holds this all together:

api/api.go
type API struct {
	handler http.Handler
	db      *gorm.DB
	config  *conf.Configuration
	mailer  mailer.Mailer
	version string
	ext     *extensions
}
1 The Router: handler manages all the incoming web requests.
2 The Storage: db connects to the database to save orders and items.
3 The Postman: mailer sends out order receipts and notifications.

Understanding this flow makes debugging "failed to charge" or "didn't receive email" bugs much easier. Here's exactly how an order travels through the system.
